## Status vocabulary

- `complete_evidenced`: a sealed artifact and focused validation support the requirement.
- `implemented_gate_pending`: code exists, but a required runtime, data, or evaluation gate is still absent.
- `blocked_prerequisite`: the implementation deliberately emits no launch authority until a named prerequisite exists.
- `not_started`: the Beads execution task remains open and no result artifact is claimed.
- `optional_not_due`: the optional stage is correctly deferred until earlier gates pass.
- `policy_enforced`: repository policy explicitly prevents a forbidden or premature action.
